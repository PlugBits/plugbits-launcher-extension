# PlugBits Launcher for kintone — UI/UX 改善提案（2026-07）

コードベース全体（サイドパネル、コンテンツスクリプト、Excel Overlay、Pro課金、設定画面、Cloudflare Worker）を精査した上での改善提案。

---

## 0. 現状の全体像

| 面 | 内容 | 起動方法 |
|---|---|---|
| サイドパネル | ショートカット / WatchList（件数） / Record Pin / Recent Records（2タブ構成、幅400px） | ツールバーアイコン |
| コマンドパレット | アプリ検索・画面遷移・管理系・開発系コマンド + 番号ショートカット1–9 | `Ctrl+/` のみ |
| Quick New Record | 一覧画面からのモーダル新規作成（スマート日付トークン対応） | `Shift+Alt+N` のみ |
| Excel Overlay | 一覧/詳細のExcelライク表示・編集（仮想スクロール、Undo/Redo、フィルハンドル、コンフリクト解決付き保存） | `Ctrl+Shift+E` +サイドパネル |
| 設定ページ | 7ペイン（全般/ショートカット/ウォッチリスト/ピン/スプレッドシート/API使用状況/Proライセンス） | 拡張オプション |
| Pro課金 | ¥980/月、Stripe + Cloudflare Worker、メールでライセンスキー送付→手動貼り付け | 外部LP |

**技術的な完成度は高い**（仮想スクロール、リビジョン競合の自動再適用、ACL評価、APIコスト最小化設計）。一方で「**作った機能にユーザーが辿り着けない**」ことが最大の課題。

---

## 1. 最重要：機能の発見性（このプロダクト最大のボトルネック）

主要3機能（パレット / Quick New / Overlay）が**すべてキーボードショートカット限定**で、画面上に入口が一切ない。さらに:

- パレット内の「ショートカットキー一覧を表示」コマンドは `'?'` キーイベントを合成発火するが、**`'?'` のハンドラがどこにも存在せず無反応（バグ）**（`content-command-palette.js:286`）。
- パレットの番号ボタン 1–9 は表示上番号があるのに**数字キーが配線されていない**。

### 提案
1. **一覧画面ヘッダに小型ランチャーボタン群を注入**（Overlay起動 / Quick New / パレット）。ホバーでショートカットキーをツールチップ表示。設定でOFF可能に。
2. **チートシートモーダルの実装**（`?` ハンドラ追加でバグ修正を兼ねる）。全ショートカットを一覧表示。
3. **番号キー 1–9 をパレット内で実際に動くように配線**。
4. サイドパネルのフッターに「キーボードショートカット」ヘルプリンク。

---

## 2. オンボーディング：権限は説明するが機能は説明しない

現状の初回体験はホスト権限モーダルのみ。新規ユーザーは空のWatchList・空のピンを見て終わる。

### 提案
1. **3ステップの機能ツアー**をサイドパネル初回表示時に（①現在のビューをWatchListに追加してみる → ②Ctrl+Shift+EでOverlay → ③Ctrl+/でパレット）。
2. **空状態にCTAを埋め込む**：「ウォッチリストはまだありません」→「＋ このビューを追加」ボタン。ピン空状態→「レコード画面で右クリック→ピン」の図解。
3. 権限オンボと機能ツアーを1つのフローに統合（許可→そのまま最初のWatchList登録まで誘導）。

---

## 3. データ保護：未保存編集が黙って消える（即修正推奨）

Overlayに `beforeunload` ガードがない。タブを閉じる/リロード/kintone内遷移で編集が無警告で消える。`hasUnsavedChanges()` は実装済みなので配線するだけ。**Pro機能（有料の編集体験）の信頼性に直結するため最優先。**

---

## 4. Pro転換ファネル：ブロックはするが売らない

- 無料ユーザーがOverlayで編集を試みると「Pro 版のみ」トーストのみ。**アップグレード導線・価格・比較が overlay 内に一切ない**。
- 購入フロー：設定→Proタブ→外部LP→購入→メール待ち→UUID手動貼り付け→Verify。長い。トライアルなし。
- **設定の「利用モード off/Standard/Pro」ラジオはランタイムで一度も読まれていないデッド設定**（`kfavExcelOverlayMode`）。「Proにしたのに編集できない」という誤解を生む。
- 無料の閲覧モードではセルが**編集可能に見える見た目**のままブロックされる（期待と現実の乖離）。
- 詳細画面の単票編集は無料、一覧の一括編集はPro——この境界がUI上どこにも説明されていない。

### 提案
1. ブロックトーストに**「Proで解放 → 詳細を見る」ボタン**を追加し、機能比較＋価格＋購入リンクのモーダルへ。
2. ツールバーの「Standard」バッジをクリック可能にして同モーダルへ。
3. **14日トライアル**（Worker側にトライアルキー発行を追加）または「1日5セルまで編集可」等の体験枠を検討。触らせてから売る。
4. デッド設定ラジオを削除、またはOverlay有効/無効トグルとして実装し直す。
5. 無料の一覧モードではセルを**明確に閲覧スタイル**（グレー背景+錠アイコン列ヘッダ）にし、境界（単票編集=無料）を設定画面とモーダルに明記。
6. ライセンスキー：purchase完了ページから `options.html?license=KEY` へのディープリンクで自動入力。

---

## 5. フィードバックの統一：4種類の通知機構が混在

Overlayトースト / サイドパネルnotice / サービスワーカーの1.2秒バッジフラッシュ / パレットは**無反応**（コピー系コマンドが成功でも失敗でも無音、`catch(_){}` で握り潰し）。

### 提案
- 共通トーストモジュールを1つ作り全面採用。特に**パレットのクリップボード系に成功/失敗トーストを必ず出す**。
- `window.prompt/confirm/alert`（お気に入り編集、ライセンス解除、権限結果等で多用）をカスタムダイアログに置換。

---

## 6. 一貫性：i18n・ダークモード・用語

**i18n の穴（設定言語が効かない箇所）:**
- コマンドパレットは**全文日本語固定**（英語ユーザーにも日本語で出る）
- `pins.js`（ピンタブ）は**全文英語固定**
- Pro ライセンスペイン、サイドパネルのタブラベル（メイン/ピン）、WatchListステータス（`refreshing...` 等）も固定
- 辞書が4ファイルに重複コピーされておりドリフト必至 → **共通辞書モジュールに一元化**

**ダークモード:**
- `launcher.css` には充実したダーク対応がある一方、**`options.css` はほぼ未対応**（設定画面がダークでも真っ白）
- launcher.css のダークブロックには実在しないクラスへのデッドCSSが大量 → 整理
- 2つのCSSでトークン値が食い違う（accent `#2563eb` vs `#0b57d0` 等）→ **共通デザイントークンファイル**を作り両方から参照

**用語:**
- 「Excel Overlay / スプレッドシート / Grid Edit」の3呼称が混在 → 1つに統一
- **「ピン」が2概念**（WatchListのピン留めと Record Pin）で同じ語・同じアイコン → Record Pin を「ブックマーク」等に改名するか、アイコンを分ける

## 7. アクセシビリティ

- パレット：`role="dialog"`/`listbox`/`option`/フォーカストラップ/aria-live が皆無
- Quick New モーダル：`role="dialog"`/`aria-modal`/フォーカストラップなし
- サイドパネルのタブ：`role=tablist` はあるが矢印キーでのタブ移動未実装
- ドラッグ&ドロップ並び替えにキーボード代替なし
- 件数バッジのエラーが `title` ツールチップのみ（スクリーンリーダー非対応）
- `prefers-reduced-motion` 未対応

Overlay本体のa11yは比較的良いので、**Overlayの水準に他サーフェスを揃える**方針で。

---

## 8. Excelとしての操作期待値（Pro価値の底上げ）

スプレッドシートユーザーの手癖で欠けているもの:
- 選択セルでの **type-to-replace**（入力開始で即上書き編集）
- **Delete/Backspace での範囲クリア**
- スケルトンローディング（現状テキスト1行のローダー）

## 9. WatchList：「監視」なのに通知しない

現状は件数の**表示**のみ（パネルを開いている間だけTTL更新）。`chrome.notifications` は未使用。

### 提案
- 項目ごとに任意の**閾値通知**（「未処理が10件を超えたらデスクトップ通知」）。alarms + 既存の件数取得を流用すれば省API設計を保てる。これは「毎朝パネルを開いて確認する」習慣を「向こうから知らせてくれる」体験に変える、**継続利用の柱**になり得る。
- サイドパネルの各エントリに**ホバーで操作ボタン**（ピン切替/編集/削除）。現状は関数はあるのにUIから呼べず、編集は設定ページ+`prompt()`頼み。

## 10. 実行ロードマップ（推奨順）

| フェーズ | 内容 | 規模 |
|---|---|---|
| **P1: 即効の修正** | beforeunload ガード / `?`チートシート実装（バグ修正）/ デッド設定ラジオ撤去 / パレットのコピー系にトースト / 番号キー配線 | 小 |
| **P2: 発見性+オンボ** | 一覧ページのランチャーボタン注入 / 空状態CTA / 3ステップツアー | 中 |
| **P3: Pro ファネル** | Overlay内アップセルモーダル / 閲覧モードの見た目修正 / トライアル / キー自動入力 | 中 |
| **P4: 基盤の一貫性** | 共通デザイントークン+optionsダークモード / i18n一元化 / a11y底上げ / 用語統一 | 大 |
| **P5: 機能進化** | WatchList閾値通知 / type-to-replace・Deleteクリア / サイドパネル内編集 | 中〜大 |

「ユーザーに使ってもらう」観点では **発見（P1/P2）→ 体験（P3）→ 信頼（P4）→ 習慣化（P5）** の順が投資対効果が高い。

## 付録：調査で見つかった明確なバグ/デッドコード

1. `content-command-palette.js:286` — ヘルプコマンドが `'?'` を合成発火するがハンドラ不在（無反応）
2. `kfavExcelOverlayMode`（off/Standard/Pro ラジオ）— 書き込まれるが誰も読まない
3. Overlay に `beforeunload` なし（`hasUnsavedChanges()` は存在）
4. `launcher.css` のダークモードブロックに実在しないクラス向けの大量デッドCSS
5. `options.css` の `.developer-pro-override-row` / `.watchlist-debug-*` に対応マークアップなし
6. パレット番号ボタン 1–9 の数字キー未配線
7. サイドパネルの `editEntry`/`togglePin`/`deleteEntry` 等がUIに未配線（設定ページ経由のみ）
